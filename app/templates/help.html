Oppsett

Sett opp {{ brand_name }} fra bunnen

Følg stegene under i rekkefølge. Du må gjøre dette kun én gang per Microsoft 365-tenant. Total tidsbruk: ca. 30 minutter.

1. Microsoft Entra ID 2. OpenAI 3. Pinecone 4. Konfigurer {{ brand_name }} 5. Verifiser Feilsøking
Steg 1

Opprett app-registrering i Microsoft Entra ID

{{ brand_name }} bruker app-only client credentials-flyten — altså en egen identitet for tjenesten, uten å låne en brukers kontekst. Dette er sikrest og krever ingen pålogging fra brukerne.

  1. Gå til entra.microsoft.com → Applications → App registrations.
  2. Klikk New registration. Gi den et navn, f.eks. {{ brand_name }} – {{ brand_owner }}.
  3. Velg Accounts in this organizational directory only. Hopp over Redirect URI.
  4. Etter opprettelse: noter ned Application (client) ID og Directory (tenant) ID.
  5. Gå til Certificates & secrets → New client secret. Velg utløpstid og kopier verdien med en gang — den vises bare én gang.
  6. Gå til API permissions → Add a permission → Microsoft Graph → Application permissions og legg til:
    • Files.Read.All
    • User.Read.All
    • Sites.Read.All (bare hvis dere vil indeksere SharePoint-områder)
  7. Klikk Grant admin consent for {{ brand_owner }}. Uten dette får app-only-kall 403.
Sikrere variant: Bruk Sites.Selected i stedet for Sites.Read.All og gi appen tilgang kun til spesifikke SharePoint-sider via en engangs PATCH mot /sites/{id}/permissions. Trinn-for-trinn i Microsofts dokumentasjon.
Steg 2

OpenAI-konto og API-nøkkel

  1. Logg inn på platform.openai.com.
  2. Gå til API keys → Create new secret key. Kopier verdien.
  3. Sjekk at kontoen har tilgang til text-embedding-3-large (alle paid-konti har det).
  4. Sett en månedlig usage limit i Limits som passer med antall dokumenter dere har.
Kostnad: ca. $0.13 per million tokens. 100 000 dokumenter på 5 sider hver ≈ 50 millioner tokens ≈ $6,50 én gang. Oppdateringer er marginale.
Steg 3

Pinecone-prosjekt og indeks

  1. Opprett konto på app.pinecone.io.
  2. Lag et nytt serverless-prosjekt (AWS, region us-east-1 er enklest).
  3. Hent API-nøkkel fra API Keys.
  4. Ikke opprett indeksen manuelt. Etter at miljøvariablene er satt, kjør: 
    python -m scripts.bootstrap_pinecone --cloud aws --region us-east-1
    Dette oppretter indeksen med nøyaktig riktig dimensjon (3072) og cosine-metric.
Steg 4

Konfigurer {{ brand_name }}

Du har to alternativer:

A) Innstillinger-UI (anbefalt)

Logg inn med admin-passordet, åpne Innstillinger, og fyll inn felt for felt. Test-knapper bekrefter at hver tjeneste svarer.

Åpne innstillinger →

B) Miljøvariabler i Railway

Sett GRAPH_*, OPENAI_*, PINECONE_*, ADMIN_PASSWORD osv. i Railway-dashboardet. Se .env.example i repoet for full liste.

Steg 5

Verifiser at alt virker

  1. Åpne Innstillinger og bruk Test tilkobling-knappene for Graph, OpenAI og Pinecone.
  2. Trykk Kjør synkronisering nå. Første runde kan ta lang tid på en stor tenant — det er normalt.
  3. Sjekk Kjøringer-fanen for å se progresjon, indekserte filer og eventuelle feil.
  4. Verifiser i Pinecone-konsollet at indeksen får vektorer.

Feilsøking

Vanlige problemer

Graph-test feiler med 403 Forbidden

Du har sannsynligvis ikke gitt admin consent for application permissions. Gå tilbake til Entra ID → API permissions og klikk Grant admin consent.

Pinecone-test feiler med "index not found"

Du må kjøre python -m scripts.bootstrap_pinecone én gang for å opprette indeksen, eller du har skrevet feil indeksnavn i innstillingene.

OpenAI-test feiler med "model not found"

Sjekk at OpenAI-kontoen har text-embedding-3-large aktivert. Trial-konti har den ikke alltid; du må ha gjort minst én betaling.

Synkronisering kjører, men ingen filer indekseres

Sjekk SYNC_SCOPE. Hvis den er users_csv uten verdi i SYNC_USERS, blir det ingen drives å lese. Sett til all_users for å hente alle.

"Delta-token utløpt"-meldinger

Helt normalt etter lang stillstand. {{ brand_name }} resetter automatisk og kjører en full re-indeksering av den drivven på neste runde.

Containeren bruker mye disk

Imaget er ~3 GB pga. Unstructured + tesseract + libreoffice. Det er normalt for self-hosted dokumentparsing. Kjør hosted Unstructured API hvis dere vil ha mindre.
